title: 'Environments'

Shows all the environments in which the service is deployed, including the cluster name, namespace, ingress status, etc.
This page also allows you to create new environments to deploy your service over. 

For instructions on how to create and edit a service environment, refer to the [Environments](/getting-started/create-service) section in the Getting Started guide.
